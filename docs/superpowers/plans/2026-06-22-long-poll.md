# Long-Poll Fix Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix `entries.php` so a GET with `?since=<ts>` that has no new data always holds the connection open until new data arrives or the timeout expires, regardless of whether the cache is valid.

**Architecture:** The long-poll loop (`sleep(2)` + `filemtime` watch) already exists in `entries.php` but is only reached when the cache is invalid. The cache-hit path returns 204 immediately when there is nothing newer than `$since`. The fix is to fall through from the cache-hit path to the long-poll loop when `$since` is set and nothing new exists. The `poll_timeout` (formerly hardcoded as `50`) moves to `infopedia.cfg` under `[entry]`.

**Tech Stack:** PHP 8+, procedural; `just e2e` / `just ci` for tests; no client-side changes needed (the client already handles 204 as "re-poll immediately" and holds no implicit delay assumption).

## Global Constraints

- CP1: Plain procedural PHP — no classes, no framework, no Composer
- Only `entries.php`, `infopedia.cfg`, `infopedia_template.cfg` change
- Config key: `poll_timeout` (integer, seconds) under `[entry]` section; default `25` if absent
- When `poll_timeout = 0`: skip the wait loop entirely (used to make tests fast)
- No client (`app2.html`) changes
- Existing behavior preserved: GET without `?since` still serves from cache immediately

Replace with:

```php
// 4. Serve from cache when valid and not forced-refresh.
//    When $since is set and cache has nothing new, fall through to the long-poll
//    wait below instead of returning 204 immediately.
if (!$refresh && isCacheValid($cache_file, $cache_max_age, $outdated_file, $cache_delay)) {
    $data = readCache($cache_file);
    $out  = _get_respond($data, $format, $since);
    if ($out !== '' || $since === '') {
        log_return(strlen($out) . ' bytes from cache');
        echo $out;
        exit;
    }
    // $since set but nothing new in cache — fall through to long-poll.
}

// 5. Long-poll: hold connection until source file changes or timeout expires.
$poll_timeout = (int)($config['poll_timeout'] ?? 25);
if ($since !== '' && $poll_timeout > 0 && file_exists($source_file)) {
    $stop_at = time() + $poll_timeout;
    while (time() < $stop_at) {
        clearstatcache(true, $source_file);
        if (filemtime($source_file) > $since_int) {
            break; // File changed — proceed to read.
        }
        sleep(2);
    }
}
```
